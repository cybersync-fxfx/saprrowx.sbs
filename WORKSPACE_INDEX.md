# Workspace Index

Indexed on 2026-04-28.

## Overview

This repository is a Sparrowx infrastructure security workspace. It combines a Node.js control server, React dashboard, Supabase-backed auth/data, remote agent tooling, guard-host scripts, threat-radar scanning, local runtime persistence, and an experimental XDP packet-drop engine.

Current major pieces:

- root Node.js + Express + WebSocket control server
- React + Vite frontend served from `frontend/dist` in production mode
- Supabase Auth, profile, agent, tunnel, and threat-radar data paths
- dynamic per-user agent installer generated by the backend
- standalone versioned agent bundle in `agent/`
- guard bootstrap, hardening, tunnel, and operator scripts
- threat-radar scanner with local intel logs and Supabase event history
- local `storage/` runtime snapshots and command ledger persistence
- eBPF/XDP blacklist engine under `xdp/`
- local Codex/Supabase skills under `.agents/skills/` and `skills/`

This pass indexed 140 non-generated files, excluding `.git/`, `node_modules/`, and `frontend/dist/`.

## Runtime Shape

1. A browser hits the panel domain and loads the React frontend from the Express server.
2. Public visitors land on the portal page and can move to login or registration.
3. Authenticated users use the dashboard over same-origin `/api/*` requests and a WebSocket connection.
4. Remote agents register, post telemetry, poll queued commands, and return command output.
5. The backend keeps live agent state in memory and periodically snapshots key state to `storage/runtime-state.json`.
6. Threat Radar scans guard-side socket/traffic state, scores IPs, logs events to Supabase, and can auto-ban through guard and connected-agent firewall paths.
7. Operator scripts can harden the guard, manage tunnels, inspect connected agents, and update agent installs.
8. The optional XDP loader compiles and attaches a BPF program that drops blacklisted IPv4 traffic at the network interface.

## XDP Engine

The `xdp/` directory contains an optional low-level packet-drop path:

- [xdp/sparrowx_xdp.c](C:/Users/cyber/Documents/GitHub/sbs.detriot/xdp/sparrowx_xdp.c): eBPF/XDP program with a `sparrowx_blacklist` hash map and IPv4 source-IP drop logic
- [xdp/load_xdp.sh](C:/Users/cyber/Documents/GitHub/sbs.detriot/xdp/load_xdp.sh): installs build dependencies, compiles the BPF object, attaches it to the default network interface, and pins `/sys/fs/bpf/sparrowx_blacklist`

This is separate from the Node.js runtime path, but it is intended to give the scanner/guard stack a faster packet-dropping layer.
